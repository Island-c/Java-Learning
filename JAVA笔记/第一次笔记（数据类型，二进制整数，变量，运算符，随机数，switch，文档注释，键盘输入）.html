数据类型

浮点数

  • int 4字节
  • float 4字节
  • 但是int是精确表示 所以表示到2^32 ，float不精确，所以是3.4x10^38
  • int a=123123123L ; 末尾加L自动转成长整型
  • float a=6.32F; 这样会丢失精度，末尾加F强制使用丢失精度的方法
  • 避免在比较中使用浮点数
  • BigDecimal，精确数字计算
  • BigInteger ，大于long（8字节）表示范围的数

char

  • unicode字符集，用两个字节表示65536个字符
  • 与C++不同，char范围是0-655355，运算时可以直接当成整数使用

boolean

  • 不要写if(is == true && done == flase)
  • 写if(is && !done)

类型转换

  • 容量小的数据可以转成容量大的字节（容量指的是表述范围，而不是几个字节。就像long是八个字节，但是表述范围还没有float四个字节的大，只是float会丢失精度）

自动转型

  • byte->short->int->long,
  • char->int->double
  • 丢失精度：int ->float , long->float , long ->double
  • int数值只要在范围内，可以转byte,short,char

强制转型

  • char a =(char)aa;

运算时类型提升问题

  • int类型与long类型相加，最后结果是long的..
  • 也可以强制转 int c=(int)(a+b)
  • int类型与double相加，结果是double

运算时溢出问题

  • long total=(long)money x year;//先把前面的转成long，然后整体提升

  • 不能写long total=(long)(money x year); //先算完就是溢出的int，再转成long也没用了

  • 纯数值时在前面加L，加在后面如果前面溢出就完了long times =70L*60*24*365*70;

前面加final，只能被赋值一次

  • 常量一般使用大写。

命名规范

  • 变量： 首字母小写，驼峰原则
  • 常量：大写字母+下划线
  • 类型：首字母大写+驼峰原则

运算符

  • 取余运算 也可以用小数

  • && 有短路，前面是false，后面就不算了

  • ++ --放在变量前后的区别

  • 位运算： 右移一位相当于除2取商；左移一位相当于乘2 （移位比直接乘除快很多）

  • 位运算： ~ 取反， &按位与 ， | 按位或 ， ^ 按位异或(相同得0,相反得1)，

    • 加号前后如果有一个是String类型，就会变成字符串相连，而不是相加

  • 三目条件运算符 x?y:z
    if (x) y ; else z;
